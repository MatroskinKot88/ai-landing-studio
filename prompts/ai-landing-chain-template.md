### Универсальны промпт для генерации AI лендингов для разных ниш ###

## 1-й Этап: Структура Лендинга ##
Ты — маркетолог и product designer в B2B.  
Контекст: Я создаю лендинг для [ОПИШИ БИЗНЕС: например, “компании, производящей холодильные витрины для кофеен”]. Целевая аудитория — [ОПИШИ ЦА: например, “владельцы кофеен и пекарен”]. Их главные боли: [ПЕРЕЧИСЛИ 2–3 БОЛИ: например, “портится выпечка, витрины выглядят дёшево, высокое энергопотребление”].  
Задача: Предложи структуру одностраничного лендинга, оптимизированного под конверсию в коммерческий запрос.  
Формат: Список секций (hero, преимущества, продукция и т.д.) с кратким описанием цели каждой секции.  
Ограничения: Не более 6 секций. Без технических деталей — только маркетинговая логика.

# Результат #
Результат → список секций: Hero, Преимущества, Продукция, Отзывы, Тарифы, CTA.

---------------------

## 2-й Этап: Продающий промпт ##
Ты — копирайтер с опытом в [НИША, например, HoReCa / retail / foodtech].  
Контекст: Структура лендинга: [ВСТАВЬ СЕКЦИИ ИЗ ЭТАПА 1]. ЦА: [ЦЕЛЕВАЯ АУДИТОРИЯ]. Боли: [БОЛИ].  
Задача: Напиши продающий контент для каждой секции:  
- Hero: заголовок (до 60 символов) + подзаголовок (до 120 символов)  
- Преимущества: 4 пункта (формат: “Результат для бизнеса — как это работает”)  
- Продукция: краткое описание 3 типов продукции  
- CTA: призыв к действию с триггером срочности или выгоды  
Формат: Markdown с заголовками секций.  
Тон: профессиональный, без жаргона, с конкретикой (цифры, временные рамки, доказательства).

# Результат #
Результат → тексты для всех секций.

----------------------

## 3-й Этап: HTML по БЭМ ##
Ты — Senior Frontend-разработчик.  
Контекст: Ниша — [НИША]. Дизайн — [СТИЛЬ: например, “премиум, тёплые тона, акцент на натуральных материалах”].  
Задача: Создай HTML-разметку секции “[НАЗВАНИЕ СЕКЦИИ, например, hero]” с использованием следующего контента:  
[ВСТАВЬ ТЕКСТ ИЗ ЭТАПА 2]  
Формат вывода: Только валидный HTML-фрагмент с корневым блоком по БЭМ (например, .hero). Без обвязки (<html>, <body>).  
Ограничения: Чистый HTML, без CSS/JS. Используй семантические теги. Все классы — по БЭМ.

# Результат #
Результат → разметка.

-----------------------

## 4-й Этап: CSS по компонентам ##
Ты — frontend-разработчик, специализирующийся на модульной вёрстке.  
Контекст: Ниша — [НИША]. Общий стиль — [ОПИСАНИЕ СТИЛЯ].  
Задача: Напиши автономный CSS для блока “[НАЗВАНИЕ БЛОКА, например, .hero]”.  
Требования:  
- Используй БЭМ-нотацию  
- Адаптивность (мобильные, планшеты)  
- Переменные CSS для цветов  
- Никаких !important, глобальных стилей  
Формат: Только CSS-код, без пояснений.  
Ограничения: Не используй фреймворки, препроцессоры.

# Результат #
Результат → стили

-----------------------

## Этап 5 (опционально): Сборка и README ##
Ты — технический писатель и разработчик.  
Задача: Создай README.md для этого лендинга. Включи:  
- Описание ниши и ЦА  
- Список компонентов  
- Инструкцию по сборке (как подключить CSS)  
- Пример использования (где вставить HTML)  
Формат: Markdown.
